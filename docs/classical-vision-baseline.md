# OpenCV 传统视觉缺陷检测基线

## 定位

该模块是确定性、可解释的传统视觉基线，适合照明、相机位置、工件姿态和表面背景较稳定的产线。它用于快速建立规则基准、过滤明显伪缺陷和辅助深度模型决策，不声称能够覆盖所有工业缺陷。

## 处理流程

1. **灰度化**：将颜色图转换为单通道亮度图，降低计算量并突出明暗差异。
2. **高斯滤波或中值滤波**：高斯滤波抑制近似高斯噪声；中值滤波更适合椒盐噪声并能较好保留边缘。
3. **CLAHE**：按局部区域增强对比度，并限制噪声被过度放大。
4. **OTSU、自适应阈值或 Canny**：OTSU 适合双峰灰度分布；自适应阈值应对缓慢变化的照明；Canny 提取局部边缘。
5. **开运算**：去除小型孤立噪声。
6. **闭运算**：连接邻近区域并填补小间隙。
7. **轮廓或连通域分析**：将前景像素聚合为候选缺陷区域。
8. **形状筛选**：按面积、长宽比和圆度过滤候选区域。圆度定义为 `4πA/P²`，越接近 1 越接近圆形。
9. **结果输出**：保存候选区域、二值图、带框轮廓图、JSON 结果和实际处理耗时。

所有参数位于 `configs/classical_vision/baseline.yaml`。阈值必须针对相机、曝光、工件尺度和缺陷成本进行校准，不能直接把默认值视为生产参数。

## 适合的工业问题

- 背景均匀、对比度明显的孔洞、污点、缺口和异物。
- 尺寸或形状有稳定公差的零件轮廓检查。
- 固定工位下明显的亮斑、暗斑、断线或连通性问题。
- 需要低延迟、可解释规则和少量样本即可上线的预筛选任务。

## 容易失败的情况

- 纹理复杂、缺陷与背景灰度接近，或缺陷主要依赖语义判断。
- 光照反射、阴影、曝光和相机姿态变化较大。
- 缺陷尺度跨度大、形态变化大或边界模糊。
- 正常纹理本身具有大量边缘，导致阈值和轮廓产生高误报。
- 多种缺陷外观相似但业务类别不同；传统形状特征通常不足以分类。

## 与深度学习组合

- 在深度模型前做图像质量门禁和 ROI 定位，减少无关区域。
- 用传统规则剔除面积、形状明显不合理的 YOLO 检测框，降低可解释的误报。
- 将 OpenCV 与 YOLO/异常检测的冲突结果送入人工复核，而不是自动覆盖模型结果。
- 用传统视觉生成候选区域以辅助标注，但最终标签仍需人工确认。
- 在模型不可用时作为明确降级策略，并在结果中保留 `detector_type=opencv_classical`，避免与模型输出混淆。

## 运行方式

```powershell
conda activate factory-vision
python -m scripts.run_classical_vision path\to\image.jpg
python -m scripts.run_classical_vision path\to\images --config configs\classical_vision\baseline.yaml
```

API：

- `POST /api/v1/classical-vision/predict`
- `POST /api/v1/classical-vision/batch`

API 当前接收服务端本地文件路径，适用于本地开发和受控产线目录。公网部署时应改为受控上传或对象存储 ID，并限制允许访问的根目录。
